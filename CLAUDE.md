# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is an OpenSCAD 3D modeling project for a custom car emblem (~120mm diameter). It replaces a Mercedes-Benz star with a mountain/landscape logo. The design is meant to be 3D printed in PETG.

## Key Files

- `emblem.scad` — main model: assembles `base()`, `dome()`, `border()`, and `logo()` modules
- `washer.scad` — standalone mounting washer (r=12mm outer, r=4.2mm inner hole)
- `pacifik_mers.stl` — imported base geometry (the star/base shape, not generated by SCAD)
- `logo.svg` / `mountain-logo.svg` — SVG mountain logo assets (SVG import is commented out in favor of the cube-based `logo2()` approach)
- `emblem.stl`, `washer.stl` — exported STL outputs for slicing
- `emblem.3mf` — 3MF project file (for PrusaSlicer / BambuStudio)
- `sideboard-label-holder_PETG_31m57s.gcode` — GCode for a separate print job (unrelated)

## Working with OpenSCAD

Render and preview with:
```bash
openscad emblem.scad          # opens GUI preview
openscad -o emblem.stl emblem.scad  # export STL from CLI
openscad -o washer.stl washer.scad
```

## Design Notes

- `$fn=200` in `emblem.scad` for smooth curves (slow to render); lower it to `$fn=50` for faster iteration
- `dome()` uses a flattened sphere (`sphere_z_factor=1/10`) clipped to a half-sphere shape
- `logo()` intersects a dome shape with `logo2()` to give the logo a curved surface
- `logo2()` is a geometric approximation of a mountain/Mercedes-star shape using three rotated cubes — the SVG-import version is commented out at line 31
- The emblem is 120mm diameter; `border()` is a 5mm-tall ring at d=120/110mm
