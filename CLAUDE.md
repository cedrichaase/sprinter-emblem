This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Purpose

A parametric OpenSCAD library for generating custom rear emblems for the Mercedes-Benz Sprinter and VW Crafter. Users supply an SVG logo; the library wraps it into a ~120mm circular emblem with a domed surface and border ring, ready to print in PETG and swap onto the vehicle.

## Source Files

- `emblem.scad` — main model, assembles `base()`, `dome()`, `border()`, and `logo()` modules
- `washer.scad` — mounting washer (r=12mm outer, r=4.2mm inner hole)
- `pacifik_mers.stl` — source base geometry (star shape, imported into `emblem.scad`; not generated by SCAD)
- `mountain-logo.svg` — example SVG input

Generated outputs (`*.stl`, `*.3mf`, `*.gcode`) are gitignored.

## Commands

```bash
openscad emblem.scad                    # open GUI preview
openscad -o emblem.stl emblem.scad      # export STL for slicing
openscad -o washer.stl washer.scad
```

Lower `$fn` from 200 to ~50 at the top of `emblem.scad` for faster iteration previews.

## Architecture

The emblem has four layers stacked at increasing Z:

1. **`base()`** — imports `pacifik_mers.stl` (the star), then adds a flat cylinder at Z=3 as a substrate
2. **`border()`** — hollow ring (d=120 outside, d=110 inside) at Z=5, 3mm tall
3. **`dome()`** — flattened sphere (`sphere_z_factor=1/10`) clipped to a half-dome; gives the face its convex curve
4. **`logo()`** — intersects the dome volume with the logo geometry so the logo inherits the curved surface

The current logo geometry (`logo2()`) is three rotated cubes forming a mountain/star shape. To use a custom SVG instead, replace `logo2()` with:

```scad
linear_extrude(height=15) scale(1/3) import(file="./your-logo.svg", center=true);
```

The SVG must be centered at the origin and sized to fit within the 120mm circle before scaling.
